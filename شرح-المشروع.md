# شرح مشروع DealTop

## ايه هو الموقع ده؟

DealTop هو موقع **أفلييت أمازون** (Amazon Affiliate Site) — يعني موقع بيعرض منتجات من أمازون وبياخد عمولة على كل عملية شراء تتم من خلال لينكاته.

الموقع متخصص في **الأجهزة الإلكترونية** وبيعرض:
- **618** لابتوب
- **345** ديسكتوب
- **63** شاشة (مونيتور)
- **11** اكسسوار
- **34** منتج آخر

**المجموع: 1,071 منتج**

---

## الداتا جاية منين؟

الداتا **مش بتتسكرب في الوقت الحقيقي**. الموقع الأصلي كان WordPress وكان بيستخدم إضافة اسمها **WP Automatic** اللي كانت بتعمل الآتي:

1. **بتسحب بيانات المنتجات من أمازون أوتوماتيك** (عنوان المنتج، الوصف، الصور، السعر)
2. بتنشرهم كمقالات (posts) على ووردبريس
3. كل مقال فيه لينك أفلييت لأمازون

### في النسخة الحالية (Next.js):

الداتا **ثابتة (Static)** — تم استخراجها من ملف النسخة الاحتياطية `.wpress` وتحويلها لملف JSON:

```
src/data/products.json    ← كل المنتجات (1,071 منتج)
src/data/pages.json       ← الصفحات الثابتة (سياسة الخصوصية، الشروط، إلخ)
```

**مفيش سكرابينج بيحصل دلوقتي** — كل البيانات مخزنة محلياً في ملفات JSON.

---

## لينكات أمازون منين؟

كل منتج فيه **Amazon Affiliate Link** بالشكل ده:

```
https://www.amazon.com/dp/B0F854GHFB?tag=a7medtamer-20
```

### تفصيل اللينك:
| الجزء | المعنى |
|-------|--------|
| `amazon.com/dp/` | صفحة المنتج على أمازون |
| `B0F854GHFB` | كود المنتج (ASIN) على أمازون |
| `?tag=a7medtamer-20` | **الأفلييت تاج** — ده اللي بيربط العمولة بحسابك في Amazon Associates |

### مصدر اللينكات:
- اللينكات دي كانت بتتولد أوتوماتيك من إضافة **WP Automatic** على ووردبريس
- الإضافة كانت بتاخد كود المنتج (ASIN) وبتضيف عليه الأفلييت تاج `a7medtamer-20`
- كمان فيه **10 لينكات ريدايركت** (`/go/1` لحد `/go/10`) بتحول لمنتجات معينة على أمازون

## ازاي الموقع بيشتغل؟

1. **كل الصفحات Static (SSG)** — بتتولد وقت البناء (build time) مش وقت الطلب
2. المستخدم بيدخل الموقع → بيشوف المنتجات → بيدوس "View on Amazon"
3. بيتحول لأمازون باللينك اللي فيه الأفلييت تاج
4. لو اشترى أي حاجة من أمازون خلال 24 ساعة → **بتاخد عمولة**

## لو عايز تضيف منتجات جديدة

حالياً لازم تضيفهم يدوي في ملف `src/data/products.json` بالشكل ده:

```json
{
  "title": "اسم المنتج",
  "slug": "product-slug",
  "date": "2026-03-28",
  "image": "https://m.media-amazon.com/images/I/XXXXX.jpg",
  "amazon_link": "https://www.amazon.com/dp/ASIN?tag=a7medtamer-20",
  "excerpt": "وصف قصير",
  "description": "وصف مفصل",
  "category": "laptops"
}
```

### لو عايز أتمتة (Automation):
ممكن تضيف سكريبت يسحب المنتجات من Amazon Product Advertising API (PA-API) ويحدث ملف products.json أوتوماتيك.

## الفرق التفصيلي بين PHP (WordPress) و Next.js

### 1. طريقة عرض الصفحات (Rendering)

**PHP/WordPress:**
```
مستخدم يطلب صفحة
  → PHP يشتغل على السيرفر
    → يسأل MySQL عن البيانات (query)
      → يبني HTML ديناميكي
        → يبعته للمتصفح
```
كل مرة حد يفتح صفحة، السيرفر بيعيد العملية دي من الأول. عشان كده كان فيه **SpeedyCache** و **LiteSpeed Cache** — بيخزنوا نسخة HTML جاهزة عشان مايعيدش الحسابات كل مرة.

**Next.js:**
```
وقت البناء (build)
  → يقرا products.json
    → يولد 1,088 صفحة HTML جاهزة
      → يرفعهم كملفات ثابتة

مستخدم يطلب صفحة
  → يستلم HTML جاهز فوراً (مفيش سيرفر، مفيش داتابيز)
```

### 2. قاعدة البيانات

**PHP/WordPress:**
```
MySQL/MariaDB (11.4)
├── wpak_posts            ← المقالات والمنتجات (3,314 سجل)
├── wpak_postmeta         ← بيانات إضافية لكل مقال
├── wpak_terms            ← التصنيفات
├── wpak_options          ← إعدادات الموقع
├── wpak_automatic_camps  ← حملات السكرابينج الأوتوماتيك
├── wpak_automatic_amazon_links ← ربط المنتجات بأمازون
├── wpak_automatic_log    ← سجل العمليات
├── wc_orders             ← طلبات WooCommerce
└── 60+ جدول تاني...
```
كل حاجة في داتابيز — المقالات، الإعدادات، المستخدمين، الطلبات، الكاش، اللوج.

**Next.js:**
```
ملفات JSON فقط
├── products.json    ← 1,071 منتج
└── pages.json       ← 6 صفحات ثابتة
```
**مفيش داتابيز خالص.** البيانات ملفات ثابتة بتتقرأ وقت البناء.

### 3. الأتمتة وسحب المنتجات

**PHP/WordPress (WP Automatic):**
```php
// Campaign Template اللي كان في قاعدة البيانات:

[product_imgs_html]   ← صور المنتج من أمازون أوتوماتيك

Price: <span style="color:#b12704">[price_with_discount]</span>
(as of [price_update_date] - Details)

<a href="[product_link]"><img src="buy-button.jpg"></a>

[gpt]
Write a detailed promotional product article of around 1000 words based on:
Title: [original_title]
Description: [product_desc]

STRICT RULES:
- Do NOT mention disadvantages
- The article must be 100% positive and promotional
[/gpt]
```

**ده كان بيحصل أوتوماتيك:**
1. WP Automatic بيدور في أمازون بـ keywords محددة
2. بيسحب بيانات المنتج (ASIN، العنوان، الوصف، الصور، السعر)
3. بيبعت الوصف لـ **GPT** يكتب مقال ترويجي 1000 كلمة
4. بينشر المقال أوتوماتيك كـ WordPress Post
5. بيضيف لينك الأفلييت (`?tag=a7medtamer-20`)

**Next.js:**
مفيش أتمتة — لو عايز تضيف منتج لازم تعدل `products.json` يدوي أو تبني سكريبت.

### 4. الإضافات (Plugins) vs الكود

**WordPress كان فيه 10 إضافات:**

| الإضافة | وظيفتها | البديل في Next.js |
|---------|---------|-------------------|
| **WP Automatic** | سحب منتجات أمازون + GPT | مفيش (بيانات ثابتة) |
| **Amazon Redirect Manager** | لينكات `/go1` - `/go10` | `src/app/go/[id]/route.ts` |
| **Ad Inserter** | إعلانات وسط المقالات | مفيش (ممكن تضيف يدوي) |
| **WooCommerce** | متجر + طلبات + دفع | مفيش (أفلييت فقط) |
| **Featured Image from URL** | صور خارجية كـ featured image | `next/image` مع remote patterns |
| **SpeedyCache** | كاشينج | مش محتاج (الصفحات ثابتة أصلاً) |
| **LiteSpeed Cache** | كاشينج متقدم | مش محتاج |
| **Classic Editor** | محرر المقالات | مفيش (مفيش لوحة تحكم) |
| **Replace Domain** | تغيير الدومين | مفيش |
| **FileOrganizer** | إدارة ملفات | مفيش |

**في Next.js بدل 10 إضافات PHP، عندك:**
- `lib/products.ts` — 45 سطر TypeScript بيعمل كل حاجة (بحث، فلترة، تصنيف)
- الباقي كله مبني في الكود مباشرة

### 5. أمن الموقع

**PHP/WordPress (.htaccess):**
```apache
# كان فيه حماية كتير في .htaccess:
Header always set X-Frame-Options "DENY"
Header always set Content-Security-Policy "frame-ancestors 'none';"
Header always set X-Content-Type-Options "nosniff"
Header always set X-XSS-Protection "1; mode=block"
Header always set X-Robots-Tag "noindex, nofollow"

# حماية لينكات الأفلييت من الدخول المباشر:
RewriteCond %{REQUEST_URI} ^/go[0-9]+/?$
RewriteCond %{HTTP_REFERER} ^$
RewriteRule ^ - [F]    ← ممنوع بدون referrer

# حماية من referrer خارجي:
RewriteCond %{HTTP_REFERER} !^https?://(www\.)?the-paramedics\.org
RewriteRule ^ - [F]    ← ممنوع من مواقع تانية
```

**Next.js:**
- مفيش `.htaccess` (مفيش Apache)
- الأمان بييجي من إن مفيش داتابيز ومفيش PHP يتهكر
- `robots.ts` بيمنع محركات البحث من `/go/`
- لو عايز حماية الريفيرر ممكن تضيفها في `middleware.ts`

### 6. طريقة التصميم (Theming)

**PHP/WordPress:**
```
themes/blogus/           ← القالب الرئيسي
├── functions.php        ← دوال القالب
├── style.css            ← الستايل
├── header.php           ← الهيدر (PHP يتنفذ على السيرفر)
├── footer.php           ← الفوتر
├── single.php           ← صفحة المقال
├── archive.php          ← صفحة الأرشيف
├── index.php            ← الصفحة الرئيسية
└── template-parts/      ← أجزاء متكررة
```
القالب PHP مخلوط مع HTML — السيرفر بينفذ الـ PHP وبيخرج HTML.

**Next.js:**
```
src/
├── app/layout.tsx       ← التصميم العام (React Component)
├── components/
│   ├── Header.tsx       ← هيدر (Client Component مع useState)
│   ├── Footer.tsx       ← فوتر (Server Component)
│   ├── ProductCard.tsx  ← كارت المنتج
│   └── Pagination.tsx   ← تنقل الصفحات
└── app/globals.css      ← Tailwind CSS
```
كل حاجة **React Components** — بتتبني وقت الـ build وبتشتغل في المتصفح.

### 7. الراوتينج (URLs)

**PHP/WordPress:**
```
the-paramedics.org/                          → index.php → WP_Query
the-paramedics.org/product/laptop-xyz/       → single-product.php → WooCommerce
the-paramedics.org/category/computers/       → archive.php → WP_Query
the-paramedics.org/go1/                      → .htaccess RewriteRule → Amazon
the-paramedics.org/privacy-policy/           → page.php → WP_Query
```
كل URL بيمر على `.htaccess` الأول، بعدين PHP بيقرر أنهي template يستخدم.

**Next.js:**
```
/                          → app/page.tsx (Static)
/product/laptop-xyz        → app/product/[slug]/page.tsx (SSG)
/category/laptops          → app/category/[slug]/page.tsx (Dynamic)
/go/1                      → app/go/[id]/route.ts (Server Route)
/privacy-policy            → app/privacy-policy/page.tsx (Static)
/search?q=dell             → app/search/page.tsx (Dynamic)
```
الراوتينج **File-based** — اسم الملف والفولدر هو الـ URL.

## ملخص الفرق

| | WordPress (PHP) | Next.js (TypeScript) |
|---|-----------|---------|
| **اللغة** | PHP 8.2 | TypeScript/React |
| **قاعدة البيانات** | MariaDB (60+ جدول) | ملفات JSON (2 ملف) |
